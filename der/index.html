Diagrama de Entidade-Relacionamento - Banco GloboTech

Este é um Diagrama de Entidade-Relacionamento que descreve a estrutura de dados de uma aplicação de análise de interações entre usuários, conteúdos e plataformas.

Legenda

  • PK - Chave Primária: identifica unicamente cada registro (linha) em uma tabela.
  • FK - Chave Estrangeira: cria ligação com outra tabela.
  • Cardinalidade: indica o número de conexões (ou registros) que podem existir entre duas tabelas.

Descrições Detalhadas das Entidades (Tabelas)

A seguir, as descrições de cada uma das tabelas do diagrama, com seus respectivos atributos:

Tabela usuario

Propósito: Esta tabela armazena informações sobre os usuários que interagem com a aplicação.

  • id_usuario (Chave Primária - PK): Um número inteiro que identifica cada usuário de forma única no sistema.

Tabela plataforma

Propósito: Esta tabela lista as diferentes plataformas onde ocorrem as interações.

  • id_plataforma (Chave Primária - PK): Um número inteiro que identifica cada plataforma de forma única.
  • nome: Um texto (varchar) de até 100 caracteres que contém o nome da plataforma.

Tabela conteudo

Propósito: Esta tabela armazena informações sobre os conteúdos que os usuários interagem, como vídeos, artigos, podcasts etc.

  • id_conteudo (Chave Primária - PK): Um número inteiro que identifica cada conteúdo de forma única.
  • nome_conteudo: Um texto (varchar) de até 255 caracteres com o nome do conteúdo.
  • tipo_conteudo: Um texto (varchar) de até 50 caracteres que especifica o tipo do conteúdo (por exemplo: "vídeo", "artigo").
  • id_plataforma (Chave Estrangeira - FK): Um número inteiro que se conecta à tabela plataforma, indicando em qual plataforma este conteúdo foi publicado.

Tabela interacao

Propósito: Esta é a tabela central que registra todas as interações dos usuários com os conteúdos. Ela conecta as outras três tabelas para registrar quem (usuário) fez o quê (tipo de interação), quando, por quanto tempo e com qual conteúdo e plataforma.

  • id_interacao (Chave Primária - PK): Um número inteiro que identifica cada interação de forma única.
  • id_usuario (Chave Estrangeira - FK): Um número inteiro que se conecta à tabela usuario, indicando qual usuário realizou a interação.
  • id_plataforma (Chave Estrangeira - FK): Um número inteiro que se conecta à tabela plataforma, indicando em qual plataforma a interação ocorreu.
  • id_conteudo (Chave Estrangeira - FK): Um número inteiro que se conecta à tabela conteudo, indicando com qual conteúdo a interação foi feita.
  • tipo_interacao: Um campo de enumeração que pode ter um dos seguintes valores: 'like', 'share' ou 'comment'.
  • data_interacao: Uma data e hora (datetime) que registra o momento em que a interação aconteceu.
  • watch_duration_seconds: Um número inteiro que registra a duração da interação em segundos.

Cardinalidades

A seguir, as descrições dos relacionamentos e suas cardinalidades entre as entidades:

  • Usuário e Interação: A cardinalidade é Um para Muitos (1:N). Isso significa que um usuário pode realizar múltiplas interações, mas cada interação está ligada a apenas um usuário.
  • Plataforma e Interação: A cardinalidade é Um para Muitos (1:N). Uma plataforma pode ter muitas interações registradas, mas cada interação ocorre em uma única plataforma.
  • Conteúdo e Interação: A cardinalidade é Um para Muitos (1:N). Um conteúdo pode ter muitas interações associadas, mas cada interação está ligada a apenas um conteúdo.
  • Plataforma e Conteúdo: A cardinalidade é Um para Muitos (1:N). Uma plataforma pode ter muitos conteúdos, mas cada conteúdo pertence a uma única plataforma.
